### 第3页 — 选题背景与研究意义

- 标题：选题背景与研究意义
- 要点：
  - 背景：高校编程教学对在线评测系统需求日益增长，但现有OJ平台存在评测效率低、功能单一、安全隔离不足等问题
  - 评测延迟：传统OJ单线程同步评测，高峰期排队严重，用户体验差
  - 功能割裂：题库、竞赛、练习分属不同子系统，学生学习数据无法互通
  - 学习反馈缺失：做题后缺少即时分析，教师无法追踪教学进度
  - 研究意义：设计并实现一个集题库管理、异步评测、竞赛训练、AI辅助学习、社区交流于一体的综合性在线评测系统
- 配图逻辑：左侧四行问题卡片（图标+短句），右侧一张"问题 → 本系统方案"的箭头对应图（纯图标，禁止真人）

### 第4页 — 需求分析

- 标题：系统需求分析
- 要点（与上页问题一一对应）：
  - 评测延迟 → 异步消息队列 + Docker沙箱隔离评测，秒级出分
  - 数据割裂 → 统一数据模型，题库/竞赛/练习/提交记录全域打通，38个Service协同
  - 学习反馈 → 多模型对话助手 + 学习数据统计看板（四维分析）
  - 安全需求 → 容器隔离 + XSS过滤 + 布隆过滤器防缓存穿透 + RBAC权限控制
- 补充：系统面向高校师生，需支持多角色（学生、教师、管理员），支持C++/Java/Python三种编程语言
- 配图逻辑：四组"需求（灰色）→ 方案（蓝色）"的对照卡片

### 第5页 — 系统总体设计

- 标题：系统总体架构设计 · 六层分治
- 要点（自上而下分层）：
  - 表现层：React 19 + Vite 7 + Ant Design 5 + TailwindCSS，Monaco Editor在线代码编辑，四通道WebSocket实时推送
  - 接口层：Spring Boot 3.2.5 RESTful API（23个Controller），JWT无状态认证 + TokenFilter请求过滤链
  - 业务层：38个Service，覆盖用户、题目、评测、竞赛、练习、社区、AI对话、通知、权限、统计全业务域
  - 评测引擎层：GoJudge + Docker沙箱，策略模式按语言分发，RabbitMQ优先级队列异步消费
  - 数据层：MySQL 8.0主从复制 + Druid连接池 + MyBatis-Plus；Redis多维缓存；Caffeine本地热点缓存
  - 存储层：MinIO对象存储（测试数据ZIP包、用户头像、代码文件）
- 配图逻辑：六层横向色块架构图，每层左侧标名称、右侧列技术图标。层间用箭头标注数据流方向

### 第8页 — 核心模块实现②：评测流水线

- 标题：异步评测流水线设计
- 要点：
  - 用户提交代码 → JudgeController校验 → RabbitMQ优先级队列（x-max-priority=10） → JudgeConsumer异步消费
  - 策略模式分发：JudgeStrategyFactory根据语言路由至CppJudgeStrategy / JavaJudgeStrategy / PythonJudgeStrategy
  - GoJudge在Docker特权容器（shm_size=256m）中编译运行，逐用例比对输出（JudgeOutputComparator）
  - 评测状态实时推送：JudgeWebSocketHandler → 前端useJudgeWebSocket Hook无刷新更新
  - 支持"测试运行"（testJudge）和"正式提交"（judge）两种模式
- 配图逻辑：横向流水线图——用户 → API → MQ Queue → Consumer → GoJudge Docker → WebSocket → 前端更新

### 第13页 — 安全体系设计

- 标题：四层纵深安全设计
- 要点：
  - 认证层：JWT无状态认证，TokenFilter统一拦截（支持Bearer Header + URL参数双通道），登出后Redis标记失效
  - 鉴权层：AOP声明式权限——@RequirePermission（AND/OR组合逻辑）、@RequireRole角色校验，RBAC细粒度权限码体系
  - 限流层：@RateLimit注解，支持IP / USER / ALL三种粒度，Redis滑动窗口计数
  - 输入安全：XssFilter + XssHttpServletRequestWrapper全链路XSS过滤，布隆过滤器防缓存穿透
- 配图逻辑：同心圆/洋葱图——从外到内：限流 → 认证 → 鉴权 → 输入过滤

### 第14页 — 性能优化

- 标题：关键性能优化策略
- 要点（每项一行，图标+短句+核心参数）：
  - MySQL主从读写分离：dynamic-datasource动态数据源，写主读从，Master 3308 / Slave 3309
  - Redis多维应用：分布式锁（tryLock）防重复提交、ZSet排行榜、验证码缓存、会话存储
  - RabbitMQ优先级队列：x-max-priority=10，prefetchCount=1逐条消费，高并发削峰
  - Caffeine本地缓存：TestCaseCacheService缓存测试用例，减少MinIO网络I/O
  - Resilience4j三级熔断：评测服务（失败率60%/60s）、AI服务（失败率40%/120s）、外部服务独立配置
  - Java 21虚拟线程：Project Loom虚拟线程处理高并发I/O，WebFlux响应式AI流式输出
- 配图逻辑：六列竖向卡片，每列顶部技术图标，中间核心参数，底部一句话收益
