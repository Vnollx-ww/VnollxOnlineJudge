# Vnollx 在线评测系统 - 系统架构文档

## 1. 文档说明

本文档描述 Vnollx Online Judge 的**系统架构**，包括整体架构、技术选型、分层设计、数据流、AI 代理架构、WebSocket、安全与扩展要点。

## 5. 数据流与关键流程

### 5.1 提交评测流程

```
用户提交代码
    → JudgeController.submit
    → 写入 Submission，发送 RabbitMQ 消息
    → 前端轮询或更常见：建立 /ws/judge 连接
    → JudgeConsumer 消费消息
    → 调用 Go-Judge 执行评测
    → 更新 Submission 状态与结果
    → JudgeWebSocketHandler 按 uid 推送结果给前端
```

### 5.2 读写分离

- **写**：走 master 数据源（提交、用户信息变更、题目/比赛管理等）。
- **读**：部分查询走 slave（如题目列表、提交列表、排名等），由动态数据源 + 注解或策略切换。

### 5.3 AI 对话流程（含 Python 代理）

```
前端 → /ws/ai 发送 JSON（type: chat, message, sessionId, modelId）
    → AiWebSocketHandler 接收
    → AiService 加载历史、拼装请求
    → ProxyAiStreamingClient 请求 Python 代理
    → Python: /v1/chat/stream/tools（SSE）
    → 代理调用对应 AI 厂商，可选执行 OjTools（查 MySQL）
    → 流式事件回传 Java
    → Java 转 WebSocket 推给前端
    → 对话落库（AiChatRecord 等）
```

详见 [AI代理架构.md](./AI代理架构.md)。

## 6. AI 代理架构（概要）

- **职责分离**：Java 负责会话、历史、鉴权；Python 负责模型调用、Function Calling、OjTools 执行。
- **协议**：Java 通过 HTTP/SSE 调用 Python `/v1/chat/stream/tools`，传入 api_key、model、messages、current_user_id 等；Python 返回 meta/content/thinking/error/done 等事件。
- **工具**：OjTools 在 Python 内直接查 MySQL（用户、题目、提交、比赛、练习等），供 AI 回答“我通过了多少题”“某题难度”等问题。
- **部署**：Python 代理可单独部署，支持国内/海外双地址配置（如 application.yml 中 domestic/overseas）。

## 7. 安全设计

- **认证**：JWT 放在 Header 或 Cookie，Filter 解析后写入上下文；未登录访问受保护接口返回 401。
- **权限**：RBAC，接口使用 @RequirePermission(PermissionCode.xxx)，AOP 根据当前用户角色查权限（含 Redis 缓存）。
- **输入**：XSS 过滤（如 XssFilter 对部分接口做请求体过滤）；SQL 使用参数化（MyBatis）。
- **密码**：BCrypt 存储；敏感接口可考虑限流。

---

## 8. 前端架构要点

- **路由**：React Router，/login、/register、/admin/*、/* 主布局；主布局内 /problems、/problem/:id、/submissions、/competitions、/practice、/user/:userId、/notifications 等。
- **API**：axios 实例 baseURL 为 `/api/v1`，请求拦截器附加 Token。
- **权限**：从后端拉取当前用户权限列表，前端用 PermissionCode 控制菜单与按钮（如 AI 对话、管理后台入口）。
- **WebSocket**：按需连接 /ws/judge、/ws/ai、/ws/notification、/ws/message，按业务订阅推送。

---

## 9. 部署架构（逻辑）

- **前端**：静态构建（Vite build），由 Nginx 托管或与后端同域；Nginx 将 /api/v1 与 /ws 反向代理到后端 8080。
- **后端**：单实例或多实例无状态部署，会话不依赖本地内存（JWT + Redis 可选）。
- **MySQL**：主从复制，应用配置主从数据源。
- **Redis**：单机或集群，缓存与锁。
- **RabbitMQ**：评测队列，消费者可水平扩展。
- **Go-Judge**：独立容器，后端通过 HTTP 调用。
- **AI 代理**：独立 Python 服务，可多实例；Java 通过配置的 URL 访问。

---

## 10. 扩展与演进

- **水平扩展**：后端、评测消费者、AI 代理均可多实例；需保证 RabbitMQ 消费与 WebSocket 会话亲和性或广播方案。
- **新语言**：在 Go-Judge 与题目配置中增加语言类型即可。
- **新 AI 厂商**：在 Python 代理中新增 Provider，Java 侧仅配置 model 与 proxy 地址。
- **新功能**：按资源增加 Controller/Service/Mapper，权限表与 PermissionCode 同步扩展。
