# 练习模块产品需求文档

## 模块定位

练习模块负责把题目组织成练习集，帮助用户按主题、难度或课程阶段进行系统训练。

## 用户角色

- **普通用户**：查看练习集、完成练习题、追踪练习进度。
- **管理员**：创建练习集、维护题目顺序和练习状态。

## 核心场景

- **练习集浏览**：用户查看可练习的专题或课程集合。
- **练习详情**：用户进入练习集查看题目列表和完成状态。
- **练习管理**：管理员创建和维护练习集。

## 功能需求

- **练习集列表**：展示标题、描述、题目数量和公开状态。
- **练习题目**：维护练习集与题目的关联和顺序。
- **进度追踪**：展示用户在练习集内的完成情况。
- **管理维护**：支持练习集创建、编辑、删除和题目绑定。

## 业务规则

- **公开控制**：未公开练习集不应展示给普通用户。
- **题目复用**：练习集复用题库题目，不复制题面和测试数据。
- **进度来源**：练习进度应基于用户提交和通过情况计算。

## 页面与交互

- **练习列表页**：展示练习集卡片和进入入口。
- **练习详情页**：展示题目列表、难度、通过状态和跳转入口。
- **管理后台练习页**：维护练习集基本信息和题目列表。

## 权限要求

- **浏览权限**：公开练习对所有用户可见。
- **提交权限**：登录用户可通过题目页提交练习题。
- **管理权限**：练习集维护仅管理员可操作。

## 非功能需求

- **一致性**：练习进度应与提交结果保持一致。
- **可维护性**：练习集题目顺序调整不能影响题目本身。
- **性能**：练习详情应避免重复查询大量提交记录。

## 验收标准

- **练习可访问**：公开练习集可以正常浏览和进入题目。
- **进度准确**：用户通过题目后，练习进度能正确更新。
- **管理可用**：管理员能维护练习集和题目关联。
