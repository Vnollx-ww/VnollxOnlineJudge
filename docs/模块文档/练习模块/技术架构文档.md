# 练习模块技术架构文档

## 模块职责

练习模块负责练习集元数据、练习题目关系和用户练习进度展示。

## 代码入口

- **用户练习接口**：`PracticeController`
- **管理练习接口**：`AdminPracticeController`
- **题目依赖接口**：`ProblemController`

## 数据模型

- **练习集表**：保存标题、描述、公开状态和排序信息。
- **练习题目关系表**：维护练习集与题目的关联和题目顺序。
- **提交记录**：用于计算用户在练习集内的完成状态。

## 核心流程

```text
管理员创建练习集
-> 绑定题目并设置顺序
-> 用户进入练习详情
-> 系统读取题目与用户提交结果
-> 展示完成进度
```

## 外部依赖

- **题目模块**：提供题目基础信息。
- **评测模块**：提供提交结果和通过状态。
- **MySQL**：保存练习集和题目关系。

## 缓存与异步

- **进度缓存**：高频访问时可缓存用户练习进度。
- **统计刷新**：可在提交完成后触发练习进度更新。

## 安全与权限

- **公开状态校验**：普通用户只能访问公开练习集。
- **管理接口鉴权**：练习维护接口需要管理员权限。

## 异常处理

- **练习不存在**：返回明确业务错误。
- **题目不存在**：绑定题目时应校验题目有效性。
- **进度计算失败**：不应影响题目列表基础展示。

## 扩展点

- **课程体系**：支持章节、知识点和学习路线。
- **推荐练习**：结合用户薄弱标签推荐练习集。
- **练习排行榜**：支持按练习集统计完成速度和正确率。
