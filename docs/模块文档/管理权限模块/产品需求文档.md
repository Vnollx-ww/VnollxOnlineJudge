# 管理权限模块产品需求文档

## 模块定位

管理权限模块负责管理后台、角色权限、菜单能力、字典配置和运营统计，是平台运营和安全控制的核心模块。

## 用户角色

- **普通管理员**：按授权范围管理题目、比赛、用户或通知。
- **超级管理员**：维护角色、权限、字典和系统级配置。

## 核心场景

- **后台管理**：管理员进入统一后台管理业务数据。
- **角色授权**：超级管理员配置角色和权限范围。
- **字典管理**：维护系统状态、枚举和展示配置。
- **运营统计**：查看用户、题目、提交、比赛等统计数据。

## 功能需求

- **后台首页**：展示关键数据、快捷入口和系统状态。
- **权限管理**：维护角色、权限点和用户角色关系。
- **字典管理**：维护业务字典项，支持启用、禁用和排序。
- **统计看板**：展示核心业务指标和趋势。
- **模块管理入口**：统一进入用户、题目、比赛、练习、通知等管理页面。

## 业务规则

- **最小权限**：管理员只能访问被授予的功能。
- **权限实时性**：权限变更后应尽快对用户生效。
- **操作可控**：高风险操作应具备确认、日志或回滚方案。

## 页面与交互

- **管理后台首页**：指标卡片、趋势图和快捷入口。
- **权限管理页**：角色列表、权限树和授权表单。
- **字典管理页**：字典分类、字典项和状态维护。
- **统计页**：按时间和业务维度查看指标。

## 权限要求

- **后台访问**：只有管理员角色可进入后台。
- **细粒度权限**：不同管理页面和操作按钮需要独立权限控制。
- **超级权限**：角色权限维护仅超级管理员可访问。

## 非功能需求

- **安全性**：防止越权访问和前端绕过。
- **可维护性**：新增管理页面应复用统一权限模型。
- **可审计性**：关键管理操作应可追踪。

## 验收标准

- **权限隔离正确**：无权限用户无法访问受限页面和接口。
- **授权生效正确**：角色权限调整后，用户可访问范围按配置变化。
- **后台入口完整**：核心业务模块都有对应管理入口。
