# 管理权限模块技术架构文档

## 模块职责

管理权限模块负责后台路由、角色权限模型、权限校验、字典配置、统计接口和管理端通用能力。

## 代码入口

- **权限管理接口**：`AdminPermissionController`
- **字典管理接口**：`AdminDictController`
- **统计接口**：`AdminStatisticsController`
- **其他管理接口**：`AdminUserController`、`AdminProblemController`、`AdminCompetitionController` 等
- **权限切面**：权限注解和权限校验切面

## 数据模型

- **用户表**：保存用户基础信息和状态。
- **角色表**：保存角色编码、名称和状态。
- **权限表**：保存接口、菜单或按钮级权限点。
- **用户角色关系表**：维护用户拥有的角色。
- **角色权限关系表**：维护角色拥有的权限。
- **字典表**：保存业务枚举和展示配置。

## 核心流程

```text
管理员登录后台
-> TokenFilter 解析用户身份
-> 权限切面读取接口权限要求
-> 查询或读取缓存中的用户权限
-> 放行或拒绝请求
```

## 外部依赖

- **用户模块**：提供管理员身份和用户状态。
- **Redis**：缓存用户权限集合和字典数据。
- **MySQL**：持久化角色、权限、字典和统计基础数据。

## 缓存与异步

- **权限缓存**：用户权限集合可缓存，减少频繁查询。
- **字典缓存**：常用字典项可缓存，提高展示性能。
- **统计聚合**：复杂统计可异步预聚合或按时间范围查询。

## 安全与权限

- **服务端校验**：权限必须在后端校验，不能依赖前端隐藏按钮。
- **接口分组**：管理接口与普通用户接口分离。
- **缓存失效**：角色或权限变更后应清理相关用户权限缓存。

## 异常处理

- **未登录**：返回认证错误。
- **无权限**：返回权限不足错误。
- **权限配置缺失**：按安全默认拒绝或记录告警。

## 扩展点

- **操作日志**：记录关键后台操作。
- **数据权限**：按组织、比赛或资源维度限制可管理范围。
- **审批流**：高风险操作接入审批机制。
