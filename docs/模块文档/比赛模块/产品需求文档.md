# 比赛模块产品需求文档

## 模块定位

比赛模块负责组织竞赛活动，覆盖比赛创建、报名、参赛、题目作答、实时排名、团队赛和防作弊治理。

## 用户角色

- **参赛用户**：报名比赛、进入比赛、提交题目、查看榜单。
- **队长**：团队赛中代表队伍参赛和承载团队统计。
- **管理员**：创建比赛、导入队伍、配置题目、查看榜单和处理作弊风险。

## 核心场景

- **个人赛**：用户报名后在比赛时间内提交题目，榜单按赛制统计。
- **团队赛**：管理员导入团队报名数据，队伍以队长账号维度参与统计。
- **比赛排名**：根据比赛规则实时或阶段性展示榜单。
- **防作弊审查**：采集比赛行为风险，管理员进行复核与导出。

## 功能需求

- **比赛管理**：创建、编辑、发布、关闭比赛，维护时间、类型和赛制。
- **题目管理**：为比赛绑定题目，配置题目顺序和展示规则。
- **报名管理**：支持个人报名和团队赛报名数据导入。
- **比赛提交**：比赛期间提交代码，校验比赛时间和参赛资格。
- **排行榜**：支持按赛制展示排名、通过数、罚时或分数。
- **防作弊**：展示风险事件、风险汇总和复核状态。

## 业务规则

- **时间约束**：比赛开始前不可提交，结束后不可继续提交。
- **资格约束**：未报名或不在队伍中的用户不能参赛提交。
- **团队赛统计**：团队赛提交统计归属队长平台账号，队员不强制关联平台账号。
- **榜单规则**：ACM 赛制按通过数和罚时排序，OI 赛制按得分排序。

## 页面与交互

- **比赛列表页**：展示比赛状态、时间和入口。
- **比赛详情页**：展示公告、题目列表、参赛信息和操作入口。
- **比赛题目页**：展示题面、编辑器和提交结果。
- **排行榜页**：展示个人或团队排名。
- **管理后台比赛页**：比赛配置、团队导入、题目绑定和防作弊查看。

## 权限要求

- **参赛权限**：参赛提交需要登录并满足比赛报名条件。
- **管理权限**：比赛创建、修改、队伍导入和防作弊复核仅管理员可操作。
- **榜单可见性**：可按比赛状态控制榜单是否公开。

## 非功能需求

- **一致性**：提交结果、榜单、比赛题目统计必须一致。
- **实时性**：ACM 榜单应尽量实时更新。
- **可靠性**：团队赛导入和排名统计必须可追溯。

## 验收标准

- **比赛流程闭环**：创建比赛、报名、提交、排名完整可用。
- **团队赛正确**：队伍导入后，队员提交能归属到队长账号统计。
- **赛制隔离**：ACM 和 OI 的提交、结算和榜单规则互不破坏。
