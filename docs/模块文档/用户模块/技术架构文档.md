# 用户模块技术架构文档

## 模块职责

用户模块负责账号、认证、邮箱验证、个人资料和用户管理能力，为其他业务模块提供用户身份和基础资料。

## 代码入口

- **用户接口**：`UserController`
- **邮箱接口**：`EmailController`
- **管理接口**：`AdminUserController`
- **认证过滤**：`TokenFilter`

## 数据模型

- **用户表**：保存账号、邮箱、密码摘要、头像、昵称、状态等信息。
- **验证码数据**：用于注册、找回密码等场景，通常需要缓存有效期。
- **用户统计**：提交数、通过数、排名等由评测、比赛和统计模块产生。

## 核心流程

```text
用户提交登录信息
-> 后端校验账号密码
-> 生成认证令牌
-> 前端保存令牌
-> 后续请求经 TokenFilter 解析用户身份
```

## 外部依赖

- **MySQL**：持久化用户资料。
- **Redis**：缓存验证码、登录态辅助数据或权限信息。
- **邮箱服务**：发送验证码和安全通知。

## 缓存与异步

- **验证码缓存**：验证码按邮箱维度设置过期时间。
- **权限缓存**：用户权限变化后应刷新或失效相关缓存。

## 安全与权限

- **密码加密**：使用不可逆摘要算法存储密码。
- **令牌鉴权**：登录后请求通过令牌识别用户。
- **接口隔离**：普通用户接口和管理接口分离。

## 异常处理

- **账号不存在**：返回统一业务错误。
- **密码错误**：不暴露过多账号状态信息。
- **验证码错误**：区分过期、错误和发送频率限制。

## 扩展点

- **第三方登录**：可扩展 OAuth 或校园统一认证。
- **安全风控**：可扩展登录失败次数限制、设备登录记录。
