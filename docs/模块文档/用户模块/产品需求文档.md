# 用户模块产品需求文档

## 模块定位

用户模块负责平台身份体系，是题目提交、比赛报名、题解互动、好友通知、管理权限等能力的基础。

## 用户角色

- **游客**：浏览公开内容，进入登录、注册、找回密码流程。
- **普通用户**：维护个人资料，提交代码，参加比赛，查看个人主页。
- **管理员**：在管理后台查询、创建、禁用或维护用户信息。

## 核心场景

- **注册登录**：用户通过账号、邮箱和密码完成注册与登录。
- **密码管理**：用户可以通过邮箱验证码找回密码。
- **个人资料**：用户维护头像、昵称、简介等展示信息。
- **个人主页**：展示用户提交、通过、题解和比赛相关信息。

## 功能需求

- **账号注册**：校验用户名、邮箱和密码规则，防止重复注册。
- **账号登录**：登录成功后返回认证令牌，前端保存登录态。
- **邮箱验证**：发送验证码，用于注册、找回密码等流程。
- **资料维护**：支持用户查看和编辑基础资料。
- **用户管理**：管理员可检索用户、调整状态和查看用户数据。

## 业务规则

- **唯一性**：用户名和邮箱应保持唯一。
- **安全性**：密码必须加密存储，不允许明文落库。
- **登录态**：需要鉴权的接口必须校验令牌有效性。
- **邮箱验证**：验证码必须设置有效期，并限制发送频率。

## 页面与交互

- **登录页**：账号密码输入、错误提示、跳转注册和找回密码。
- **注册页**：用户信息填写、邮箱校验、注册结果反馈。
- **设置页**：个人资料编辑、保存状态反馈。
- **用户主页**：用户公开信息和数据概览。

## 权限要求

- **本人权限**：用户只能修改自己的资料和密码。
- **管理权限**：用户管理能力仅管理员可访问。

## 非功能需求

- **安全**：防止暴力登录、验证码滥用和越权修改。
- **可用性**：登录注册失败时应提供明确错误提示。
- **一致性**：用户状态变化后，相关鉴权与权限缓存应及时生效。

## 验收标准

- **注册登录可用**：正常注册、登录、退出、找回密码流程可完成。
- **异常提示明确**：重复用户名、验证码错误、密码错误都有明确提示。
- **越权被拦截**：普通用户不能修改他人资料或访问管理接口。
