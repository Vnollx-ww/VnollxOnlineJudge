# 题目模块产品需求文档

## 模块定位

题目模块负责题库内容管理和题目展示，是练习、比赛、评测、题解和智能助手分析的基础数据来源。

## 用户角色

- **普通用户**：浏览题目、筛选题目、查看题面并提交代码。
- **管理员**：创建、编辑、下线题目，维护标签、难度和测试数据。

## 核心场景

- **题目列表**：按难度、标签、关键词筛选题目。
- **题目详情**：查看题面、输入输出格式、样例、限制和提交入口。
- **题目管理**：管理员维护题面、测试数据、标签和公开状态。

## 功能需求

- **题目检索**：支持分页、关键字、难度、标签等筛选。
- **题面展示**：支持 Markdown 内容、样例和评测限制展示。
- **标签管理**：支持题目标签分类，便于检索和统计。
- **测试数据管理**：支持上传、替换和校验测试数据。
- **题目类型**：支持标准题、构造题、浮点误差等评测配置。

## 业务规则

- **公开控制**：未公开题目不得在普通题库列表展示。
- **数据完整**：可提交题目必须具备必要的评测配置和测试数据。
- **标签一致**：题目标签应使用统一标签表，不建议自由文本散落。

## 页面与交互

- **题目列表页**：搜索、筛选、分页、题目状态展示。
- **题目详情页**：题面阅读、代码编辑器、提交按钮、题解入口。
- **管理后台题目页**：题目表单、标签选择、测试数据上传。

## 权限要求

- **浏览权限**：公开题目对所有用户可见。
- **提交权限**：登录用户可提交普通题目。
- **管理权限**：题目新增、编辑、删除、数据上传仅管理员可操作。

## 非功能需求

- **可读性**：题面渲染应稳定支持代码块、表格和公式类内容。
- **可靠性**：测试数据替换不能破坏已上线题目的评测流程。
- **性能**：题目列表分页查询应避免一次性加载过多内容。

## 验收标准

- **题目可检索**：关键字、标签和难度筛选结果准确。
- **题面可提交**：公开且配置完整的题目可以进入评测流程。
- **权限隔离正确**：普通用户不能访问题目管理能力。
