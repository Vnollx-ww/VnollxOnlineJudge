# 题目模块技术架构文档

## 模块职责

题目模块提供题目基础数据、题面内容、标签分类、评测参数和测试数据对象管理。

## 代码入口

- **用户题目接口**：`ProblemController`
- **管理题目接口**：`AdminProblemController`
- **标签接口**：`TagController`、`AdminTagController`

## 数据模型

- **题目表**：保存题目标题、题面、难度、限制、状态和评测配置。
- **标签表**：保存标签名称、颜色或分类信息。
- **题目标签关系**：维护题目与标签的多对多关系。
- **测试数据对象**：通常存储在对象存储中，通过题目编号关联。

## 核心流程

```text
管理员创建题目
-> 填写题面与评测参数
-> 绑定标签
-> 上传测试数据
-> 发布题目
-> 用户浏览并提交
```

## 外部依赖

- **MySQL**：保存题目元数据和标签关系。
- **MinIO**：保存测试数据压缩包、特殊评测程序等对象。
- **评测模块**：读取题目限制、测试数据和评测配置。

## 缓存与异步

- **列表缓存**：热门题目、标签列表可按需缓存。
- **对象同步**：多评测节点场景下，测试数据需要被评测机可访问或预热。

## 安全与权限

- **管理鉴权**：题目管理接口必须校验管理员权限。
- **文件校验**：测试数据上传应校验文件类型、大小和结构。
- **公开状态**：查询题目时必须区分普通用户和管理员视角。

## 异常处理

- **题目不存在**：返回明确业务错误。
- **测试数据缺失**：提交时阻止进入评测或返回配置错误。
- **对象存储失败**：上传失败需要回滚或提示重新上传。

## 扩展点

- **题目版本**：支持题面和测试数据版本管理。
- **题目审核**：支持投稿、审核、发布流程。
- **高级评测配置**：支持子任务、特殊比较器和交互题。
