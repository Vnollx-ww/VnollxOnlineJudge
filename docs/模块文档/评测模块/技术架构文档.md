# 评测模块技术架构文档

## 模块职责

评测模块负责提交入库、消息投递、评测机调度、沙箱执行、结果聚合和实时通知。

## 代码入口

- **提交接口**：`JudgeController`、`SubmissionController`
- **评测服务**：`JudgeService`
- **提交服务**：`SubmissionService`
- **消息生产与消费**：评测生产者、评测消费者
- **评测代理**：`judge-agent`

## 数据模型

- **提交记录**：保存代码、语言、状态、耗时、内存、错误信息、测试点统计。
- **评测消息**：携带提交编号、题目编号、语言、代码、比赛上下文等信息。
- **评测结果**：包含编译结果、运行结果、输出比较结果和最终状态。

## 核心流程

```text
用户提交代码
-> 后端校验题目、语言、权限和比赛状态
-> 保存提交记录为等待评测
-> 投递评测消息到 RabbitMQ
-> 消费者选择 Judge Agent
-> Agent 调用本机沙箱执行
-> 返回评测结果
-> 更新提交记录和相关统计
-> WebSocket 推送结果
```

## 外部依赖

- **RabbitMQ**：承载异步评测队列。
- **Judge Agent**：评测业务代理，负责本机测试数据读取和沙箱调用。
- **go-judge**：代码执行沙箱。
- **MinIO**：保存题目测试数据。
- **WebSocket**：推送评测进度和结果。

## 缓存与异步

- **队列异步化**：提交请求不直接执行用户代码。
- **评测节点容量**：按评测节点容量进行调度和限流。
- **测试数据预热**：评测机可在启动时预热对象存储中的测试数据。

## 安全与权限

- **沙箱隔离**：限制进程、文件、网络、时间和内存。
- **提交校验**：后端校验语言、题目状态、比赛状态和用户身份。
- **错误隔离**：评测机异常不应拖垮主后端服务。

## 异常处理

- **编译失败**：记录编译错误并返回对应状态。
- **运行异常**：记录运行错误、超时或内存超限。
- **评测机不可用**：触发节点降级、重试或返回系统错误。
- **消息重复**：结果更新应尽量具备幂等能力。

## 扩展点

- **多赛制评测**：支持 ACM、OI、IOI 等不同聚合策略。
- **多评测集群**：支持按区域、容量或语言路由到不同评测机。
- **特殊评测**：支持自定义比较器、交互题和子任务。
