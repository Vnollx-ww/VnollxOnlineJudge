# 智能助手模块产品需求文档

## 模块定位

智能助手模块负责为用户提供编程学习辅助，包括多模型对话、题目分析、提交诊断、学习建议和管理端模型配置。

## 用户角色

- **普通用户**：与智能助手对话，获取题目理解、代码思路和学习建议。
- **管理员**：配置模型、查看模型状态和维护供应商参数。

## 核心场景

- **流式对话**：用户输入问题后，智能助手以流式方式返回回答。
- **题目辅助**：围绕题目提供思路提示、复杂度分析和易错点。
- **学习诊断**：结合用户提交记录和题目分布生成学习建议。
- **模型管理**：管理员维护模型启停、优先级和供应商配置。

## 功能需求

- **多模型接入**：支持多个模型供应商和模型切换。
- **流式输出**：回答过程需要逐步展示，减少等待感。
- **工具调用**：助手可查询用户提交、题目分布等平台数据。
- **会话上下文**：支持连续对话和上下文传递。
- **模型配置**：后台支持模型新增、编辑、启用和禁用。

## 业务规则

- **权限边界**：智能助手只能访问当前用户被授权的数据。
- **敏感信息**：不得暴露密钥、系统配置和其他用户隐私。
- **模型降级**：首选模型不可用时应具备备用模型或明确错误提示。

## 页面与交互

- **助手入口**：用户在题目、个人学习或独立入口中发起对话。
- **流式消息**：前端逐字或分段展示回答内容。
- **管理后台模型页**：展示模型列表、状态和配置表单。

## 权限要求

- **用户鉴权**：涉及个人数据的工具调用必须登录。
- **管理权限**：模型配置仅管理员可操作。

## 非功能需求

- **稳定性**：模型供应商异常不能影响主业务链路。
- **可观测性**：记录请求状态、耗时和错误原因。
- **安全性**：密钥只能保存在服务端安全配置中。

## 验收标准

- **对话可用**：用户能正常发起流式对话并收到回答。
- **工具可用**：助手能按授权查询当前用户的学习数据。
- **配置生效**：管理员修改模型配置后调用链路按预期变化。
