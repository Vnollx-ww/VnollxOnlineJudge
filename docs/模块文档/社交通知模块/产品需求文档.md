# 社交通知模块产品需求文档

## 模块定位

社交通知模块负责用户之间的关系、私信互动和系统消息提醒，增强平台的协作与反馈能力。

## 用户角色

- **普通用户**：添加好友、查看通知、阅读私信。
- **管理员**：发送系统通知，维护通知内容和状态。

## 核心场景

- **好友关系**：用户发起好友申请，对方同意或拒绝。
- **私信沟通**：好友之间或允许范围内发送私信。
- **系统通知**：平台向用户推送比赛、评测、系统公告等消息。

## 功能需求

- **好友申请**：支持申请、同意、拒绝和删除好友。
- **私信消息**：支持会话列表、消息详情和未读状态。
- **通知中心**：展示系统通知、业务通知和已读状态。
- **管理通知**：管理员创建、发送和管理通知。

## 业务规则

- **关系唯一**：两个用户之间的好友关系应唯一。
- **未读统计**：消息阅读后应更新未读数量。
- **通知范围**：系统通知可面向全体用户或指定用户。

## 页面与交互

- **好友页**：展示好友列表、申请列表和操作按钮。
- **通知列表页**：展示通知类型、标题、时间和已读状态。
- **通知详情页**：展示通知正文和业务跳转入口。
- **管理后台通知页**：创建和管理通知。

## 权限要求

- **本人可见**：用户只能查看自己的好友、私信和通知。
- **管理权限**：系统通知创建和发送仅管理员可操作。

## 非功能需求

- **及时性**：重要通知应尽量实时或准实时到达。
- **一致性**：未读数量和消息状态应准确。
- **可追溯**：系统通知应保留发送记录。

## 验收标准

- **好友流程可用**：申请、同意、拒绝、删除状态正确。
- **通知状态正确**：未读、已读和详情展示准确。
- **权限隔离正确**：用户不能读取他人的通知和私信。
