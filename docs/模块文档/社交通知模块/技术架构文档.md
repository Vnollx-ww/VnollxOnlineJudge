# 社交通知模块技术架构文档

## 模块职责

社交通知模块负责好友关系、私信会话、系统通知和消息已读状态。

## 代码入口

- **好友接口**：`FriendController`
- **通知接口**：`NotificationController`
- **管理通知接口**：`AdminNotificationController`

## 数据模型

- **好友关系表**：保存申请方、接收方、关系状态和时间。
- **消息表**：保存发送方、接收方、内容、已读状态和会话信息。
- **通知表**：保存通知标题、内容、类型、目标用户和已读状态。

## 核心流程

```text
用户发起好友申请
-> 后端校验目标用户和重复关系
-> 保存申请状态
-> 对方处理申请
-> 状态变更为好友或拒绝
```

```text
管理员创建系统通知
-> 选择发送范围
-> 保存通知记录
-> 用户通知中心读取
-> 用户打开后标记已读
```

## 外部依赖

- **用户模块**：校验用户身份和目标用户。
- **MySQL**：保存好友、消息和通知记录。
- **WebSocket**：可用于实时通知推送。

## 缓存与异步

- **未读数量缓存**：高频展示时可缓存未读数。
- **批量通知发送**：全站通知可异步生成用户通知记录。

## 安全与权限

- **数据归属校验**：查询和修改通知、私信必须校验当前用户。
- **管理接口鉴权**：系统通知管理需要管理员权限。
- **内容过滤**：私信和通知内容可接入敏感词或审核机制。

## 异常处理

- **目标用户不存在**：好友申请或私信发送失败。
- **重复申请**：返回当前关系状态。
- **通知不存在**：详情页返回明确错误。

## 扩展点

- **实时在线状态**：展示好友在线状态。
- **消息撤回**：支持私信撤回和删除。
- **通知订阅**：用户自定义通知类型订阅。
