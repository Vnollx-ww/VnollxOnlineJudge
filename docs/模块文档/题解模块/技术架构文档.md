# 题解模块技术架构文档

## 模块职责

题解模块负责题解内容、评论、点赞和题目关联关系的管理。

## 代码入口

- **题解接口**：`SolveController`
- **管理题解接口**：`AdminSolveController`
- **评论接口**：`CommentController`

## 数据模型

- **题解表**：保存标题、正文、作者、题目编号、状态和统计信息。
- **评论表**：保存评论内容、作者、关联题解和层级关系。
- **点赞记录**：保存用户对题解或评论的点赞状态。

## 核心流程

```text
用户选择题目发布题解
-> 后端校验登录态和题目有效性
-> 保存题解内容
-> 列表和详情页读取题解
-> 用户评论或点赞
-> 更新互动统计
```

## 外部依赖

- **题目模块**：校验题解关联题目。
- **用户模块**：提供作者信息。
- **MySQL**：保存题解、评论和互动记录。

## 缓存与异步

- **热度统计**：点赞数、评论数可缓存或异步更新。
- **内容审核**：可扩展异步审核队列。

## 安全与权限

- **内容安全**：Markdown 渲染前后都应注意脚本注入风险。
- **作者校验**：编辑和删除必须校验作者或管理员权限。
- **频率限制**：发布、评论、点赞可按需增加频率限制。

## 异常处理

- **题目不存在**：阻止发布。
- **题解不存在**：详情页返回明确错误。
- **重复点赞**：点赞接口应保证幂等。

## 扩展点

- **草稿箱**：支持题解草稿保存。
- **审核流**：支持题解发布前审核。
- **收藏功能**：支持收藏题解和学习清单。
