## 2. 产品概述

### 2.1 产品定位

Vnollx 在线评测系统是一个面向算法学习与竞赛的**在线判题平台**，为程序员提供题目练习、多语言代码提交、实时评测、竞赛管理、题解分享、社交与 AI 学习建议等一体化能力。

### 2.2 目标用户

| 用户类型 | 描述 | 典型场景 |
|----------|------|----------|
| 学生/自学者 | 刷题、备赛、提升编程能力 | 每日练习、按标签/难度刷题、查看题解 |
| 教师/管理员 | 出题、办赛、管理用户与内容 | 题目/比赛/练习管理、权限与统计 |
| 竞赛选手 | 参加限时算法竞赛 | 比赛报名、赛中提交、实时排名 |
| 社区用户 | 交流与分享 | 题解发布、评论、好友与私信、通知 |

### 2.3 核心价值

- **一站式练习**：题目浏览、在线编码、即时评测、历史记录与学习统计。
- **多语言支持**：C++、Python、Java 等，统一评测引擎与限制（时间/内存）。
- **竞赛能力**：公开/私有比赛、ACM 赛制、实时排名与榜单。
- **社区与成长**：题解、评论、好友、私信、系统通知与个人学习数据。
- **AI 辅助**：多模型 AI 对话、题目分析、学习建议（含个人主页入口）。

## 5. 非功能需求

### 5.1 性能

- 题目列表、提交列表等列表接口支持分页，响应时间建议 &lt; 500ms（常规数据量）。
- 评测任务异步处理，提交后快速返回，结果通过 WebSocket 推送，避免长轮询。
- 支持 MySQL 主从、Redis 缓存、连接池等，以支撑一定并发。

### 5.2 可用性与扩展

- 核心服务可容器化部署（Docker），便于水平扩展。
- 依赖组件（MySQL、Redis、RabbitMQ、MinIO、Go-Judge）可单独部署与扩容。

### 5.3 安全

- 认证：JWT，密钥可配置，过期时间可配置。
- 密码：BCrypt 等不可逆加密存储。
- 接口：需登录的接口统一鉴权；敏感操作需校验权限。
- 输入：防 SQL 注入、XSS；文件上传类型与大小限制。

### 5.4 运维与监控

- 健康检查：Spring Boot Actuator `/actuator/health`。
- 指标：Prometheus 格式指标暴露，便于监控与告警。
- 日志：关键操作与异常有日志可追溯。

### 5.5 兼容性

- 后端：Java 21+，Spring Boot 3.x。  
- 前端：现代浏览器（Chrome、Firefox、Edge、Safari 等），支持 ES 与常用 Web API。  
- 移动端：以响应式为主，不单独承诺原生 App。
