## 维护规则

- **先归类再编写**：新增内容必须先判断属于总览、模块、运维、规划还是归档。
- **模块内成对维护**：业务模块至少包含 `产品需求文档.md` 和 `技术架构文档.md`。
- **总览不写细节**：跨模块共性放在总览，具体流程、接口、表结构、异常处理放到模块文档。
- **历史资料进归档**：论文、旧方案、临时调研等不再作为主导航入口的文档放入 `归档`。
