# Vnollx 在线评测系统 - 部署文档

## 1. 文档说明

本文档说明 **Vnollx Online Judge** 的部署方式，包括环境要求、Docker 一键部署、手动部署、Nginx 配置、AI 代理部署、监控与常见问题。

---

## 2. 环境要求

### 2.1 服务器与系统

| 项目 | 要求 |
|------|------|
| 系统 | Linux（推荐 Ubuntu 20.04+ / CentOS 7+）或 Windows Server |
| CPU | 建议 4 核及以上（评测与 AI 并发时） |
| 内存 | 建议 8GB 及以上（MySQL + Redis + 后端 + 评测） |
| 磁盘 | 建议 50GB+ SSD（数据库、MinIO、日志） |
| 网络 | 可访问外网（拉取镜像、AI API）；如需公网访问需开放端口 |

### 2.2 软件依赖（Docker 部署时由镜像提供）

| 组件 | 版本/说明 |

| Docker | 20.10+ |
| Docker Compose | 2.0+（推荐 v2） |
| （可选）Git | 用于克隆代码 |

## 3. Docker 一键部署（推荐）

### 3.1 使用现成镜像（仅后端与中间件）

当前仓库 `docker-compose.yml` 使用镜像 `vnollx/vnollx:latest` 启动后端，并搭配 MySQL 主从、Redis、RabbitMQ、MinIO、Go-Judge。

**步骤：**

```bash
# 1. 克隆项目
git clone https://github.com/Vnollx-ww/VnollxOnlineJudge.git
cd VnollxOnlineJudge

# 2. 启动所有服务（后端依赖的中间件 + 后端）
docker-compose up -d

# 3. 查看状态
docker-compose ps
```

**服务与端口：**

| 服务 | 容器名 | 端口映射 | 说明 |
|------|--------|----------|------|
| 后端 | vnollx | 8080:8080 | Spring Boot 应用 |
| MySQL 主库 | mysql-master | 3308:3308 | 主库 |
| MySQL 从库 | mysql-slave | 3309:3309 | 从库 |
| Redis | redis | 内部 6379 | 无端口映射时可仅内网访问 |
| RabbitMQ | rabbitmq | 5672, 15672 | 15672 为管理界面 |
| MinIO | minio | 9000, 9001 | 9001 为控制台 |
| Go-Judge | go-judge | 5050:5050 | 评测服务 |

**首次部署注意：**

- 若使用空白数据库，需执行建表与初始数据 SQL（见 5.2 节）。
- 后端镜像内通常已包含默认配置，指向上述容器名（如 `mysql-master`、`redis`、`rabbitmq`）。

### 3.2 本地构建后端镜像再编排

若需从源码构建后端镜像并编排：

```bash
# 在项目根目录
docker build -t vnollx/vnollx:local .

# 修改 docker-compose.yml 中 vnollx 的 image 为 vnollx/vnollx:local 后
docker-compose up -d
```

### 3.3 前端单独部署（Nginx 容器）

前端需单独构建并挂 Nginx，与后端通过 Nginx 反向代理对接。

**构建前端镜像：**

```bash
cd frontend
docker build -t vnollx-web:latest .
```

**运行前端容器（需与后端同网段，且 Nginx 配置中 proxy_pass 指向后端）：**

```bash
docker run -d --name vnollx-web --network app-network -p 3000:3000 vnollx-web:latest
```

确保 `frontend/nginx.conf` 中：

- `location /api/v1/` 的 `proxy_pass` 指向后端（如 `http://vnollx:8080`）。
- `location /ws/` 的 `proxy_pass` 同样指向后端。

若后端与前端不在同一 Docker 网络，需将 `vnollx` 改为实际后端地址（如宿主机 IP 或域名）。

## 4. 数据库初始化

### 4.1 创建数据库与用户

```sql
CREATE DATABASE vnollxonlinejudge
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

### 4.2 导入表结构与初始数据

项目内 SQL 文件路径示例：`src/main/resources/db/vnollxonlinejudge.sql`。

**Docker 环境（主库端口 3308）：**

```bash
docker exec -i mysql-master mysql -uroot -prootpassword vnollxonlinejudge < src/main/resources/db/vnollxonlinejudge.sql
```

**本机 MySQL：**

```bash
mysql -u root -p vnollxonlinejudge < src/main/resources/db/vnollxonlinejudge.sql
```

### 4.3 MySQL 主从复制（docker-compose 已配）

- 主库：`mysql-master`，端口 3308，已开启 binlog、GTID。
- 从库：`mysql-slave`，端口 3309，复制 `vnollxonlinejudge`。
- 若从库未自动同步，需在从库执行 `CHANGE MASTER TO` 并 `START SLAVE`（用户/密码与 compose 中一致）。

## 5. 手动部署（非 Docker）

### 5.1 后端

```bash
# 1. 编译
mvn clean package -DskipTests

# 2. 修改配置
# 编辑 src/main/resources/application.yml：
# - 数据库 URL、用户名、密码（主/从）
# - Redis host/port
# - RabbitMQ host/port/用户名/密码
# - MinIO 端点与密钥
# - Go-Judge 地址（如 http://localhost:5050）
# - AI 代理地址（如 http://localhost:8000）

# 3. 启动
java -jar target/VnollxOnlineJudge-*.jar
```

默认端口：**8080**。

### 5.2 前端

```bash
cd frontend
npm install
npm run build
```

将 `dist` 目录放到 Nginx 根目录（见下节），或使用 `npm run dev` 仅做开发（需配置代理到后端）。

### 5.3 Go-Judge

- 从 [criyle/go-judge](https://github.com/criyle/go-judge) 获取可执行文件或 Docker 镜像。
- 保证后端能访问其 HTTP 接口（默认 5050）。

### 5.4 MinIO

- 创建 Bucket（如 `vnollx`），用于头像、题目数据等。
- 在应用配置中填写 endpoint、accessKey、secretKey、bucket。

### 5.5 RabbitMQ

- 创建用户与虚拟主机（如需）。
- 应用配置中指定 host、port、username、password、virtualHost；队列由应用自动声明。

## 6. Nginx 配置

### 6.1 与前端一起部署（生产常见）

前端构建产物由 Nginx 托管，并反向代理 `/api/v1` 与 `/ws` 到后端。

**示例（与仓库中 frontend/nginx.conf 一致）：**

```nginx
server {
    listen 3000;
    server_name localhost;
    root /usr/share/nginx/html;
    client_max_body_size 50M;

    gzip on;
    gzip_vary on;
    gzip_types text/plain text/css application/json application/javascript;

    # 后端 API
    location /api/v1/ {
        proxy_pass http://vnollx:8080;   # Docker 内用服务名；本机可写 http://127.0.0.1:8080
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # WebSocket
    location /ws/ {
        proxy_pass http://vnollx:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }

    # 静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        try_files $uri =404;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    location = /index.html {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

若使用 80/443，将 `listen 3000` 改为 `listen 80` 或 `listen 443 ssl`，并配置 SSL 证书。

### 6.2 仅反向代理（前端另起）

若前端与 Nginx 分离（如前端单独一个域名或端口），可在同一 Nginx 中增加一个 `server`，只做 `/api/v1` 与 `/ws` 的 `proxy_pass` 到后端 8080，配置要点同上。

## 7. AI 代理部署（Python）

AI 对话依赖独立 **Python 代理** 服务，Java 通过 HTTP/SSE 调用。

### 7.1 代理项目位置与依赖

- 代理代码在 **VnollxOnlineJudge-Proxy** 目录（与主项目同层级或按实际路径）。
- 安装依赖：`pip install -r requirements.txt`（含 FastAPI、uvicorn、数据库驱动等）。

### 7.2 环境变量

代理需连接 OJ 的 MySQL（只读即可），用于工具调用查库。示例：

```bash
DB_HOST=mysql-master    # 或 127.0.0.1
DB_PORT=3308
DB_USER=root
DB_PASSWORD=rootpassword
DB_NAME=vnollxonlinejudge
```

### 7.3 启动

```bash
cd VnollxOnlineJudge-Proxy
uvicorn main:app --host 0.0.0.0 --port 8000
```

默认端口：**8000**。

### 7.4 Java 侧配置

在 `application.yml` 中配置代理地址：

```yaml
ai:
  proxy:
    domestic:
      url: http://111.230.105.54:8000   # 国内
    overseas:
      url: http://74.122.24.92:8000     # 海外
    timeout: 300
```

按实际部署修改为可访问的代理 URL（如 `http://ai-proxy:8000` 或 `http://主机:8000`）。

### 7.5 Docker 中运行代理（示例）

```yaml
# 在 docker-compose.yml 中追加
services:
  ai-proxy:
    build: ./VnollxOnlineJudge-Proxy
    ports:
      - "8000:8000"
    environment:
      - DB_HOST=mysql-master
      - DB_PORT=3308
      - DB_USER=root
      - DB_PASSWORD=rootpassword
      - DB_NAME=vnollxonlinejudge
    depends_on:
      - mysql-master
    networks:
      - app-network
```

后端 `ai.proxy.domestic.url` 改为 `http://ai-proxy:8000` 即可。

## 8. 自动化部署脚本（vnollx-deploy.py）

项目根目录提供 **vnollx-deploy.py**，用于从本机构建镜像、推送到 Docker Hub，再通过 SSH 在远程服务器拉取并重启容器。

### 8.1 环境准备

- 本机：已安装 Docker、Python3，以及 `paramiko`、`dotenv`。
- 远程：已安装 Docker，且本机可通过 SSH 密钥登录（如 `~/.ssh/id_ed25519`）。

### 8.2 配置

在项目根目录创建 `.env`：

```ini
DOCKERHUB_USER=你的DockerHub用户名
REMOTE_IP=目标服务器IP
REMOTE_USER=SSH登录用户名
```

### 8.3 使用

```bash
python vnollx-deploy.py
```

按提示选择部署目标（1=后端，2=前端），输入版本号；脚本会构建、打 tag、推送、SSH 到远程执行拉取与重启。前端/后端需在远程已加入同一 Docker 网络（如 `app-network`），或自行修改 Nginx 中的 upstream 地址。

## 9. 监控与健康检查

### 9.1 Spring Boot Actuator

- 健康：`GET /actuator/health`
- 指标：`GET /actuator/metrics`、Prometheus：`GET /actuator/prometheus`
- 需在内网或通过 Nginx 限制访问，避免暴露到公网。

### 9.2 Prometheus（可选）

在 Prometheus 中配置抓取目标为 `http://后端:8080/actuator/prometheus`，再配合 Grafana 做看板。

### 9.3 日志

- 日志路径与格式由 `logback-spring.xml` 或默认配置决定；容器内可将日志挂载到宿主机便于收集。
- 生产建议按日期滚动、保留天数限制。

## 10. 常见问题与排查

### 10.1 后端无法连接 MySQL

- 检查 `application.yml` 中 host/port 与真实一致（Docker 内用服务名，本机用 127.0.0.1）。
- 确认数据库已创建、用户有权限、防火墙未拦截 3306/3308。

### 10.2 评测无结果或一直 Pending

- 确认 RabbitMQ 正常、消费者在运行；查看队列是否有积压。
- 确认 Go-Judge 进程/容器在运行且端口 5050 可达；检查后端配置的 Go-Judge URL。

### 10.3 WebSocket 连接失败

- Nginx 必须正确设置 `Upgrade`、`Connection`，且 `proxy_read_timeout` 足够大（如 86400）。
- 若用 HTTPS，确保 `X-Forwarded-Proto` 为 `https`，避免前端用 wss 而后端认为非安全。

### 10.4 AI 对话超时或不可用

- 确认 Python 代理已启动且 Java 配置的 URL 能访问（本机 curl 测试）。
- 检查代理日志、网络与 AI 厂商 API Key/配额；Resilience4j 熔断打开时也会不可用，需等待恢复或重启。

### 10.5 前端访问 404（刷新或直接打开子路径）

- 必须使用 `try_files $uri $uri/ /index.html;`，保证 SPA 路由由前端接管。

### 10.6 主从延迟

- 从库延迟较大时，刚提交的数据在从库查可能查不到；可对强一致读走主库或接受短暂延迟。
